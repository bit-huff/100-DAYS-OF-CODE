CSS ANIMATION PROPERTY

The CSS animation property is used to animate elements by applying a sequence of keyframes, which define how the element's styles change at different points during the animation. Unlike the transition property, which only animates between two states, the animation property allows you to create more complex sequences with multiple steps and finer control over the timing, direction, and repetition.

Syntax:- 


        
            element {
                animation: name duration timing-function delay iteraction-count direction fill-mode;
            }

Components of Animation

  • name: The name of the @keyframes at-rule that defines the animation. It connects the keyframes to the element's animation.
  • duration:Specifies how long one cycle of the animation should take. It can be defined in seconds (s) or milliseconds (ms).
  • timing-function: Specifies the speed curve of the animation. Common values are the same as with transition: ease, linear, ease-in, ease-out, ease-in-out, or custom curves using cubic-bezier().
  • delay: Defines a delay before the animation starts, in seconds or milliseconds.
  • iteration-count: Defines how many times the animation should repeat. Common values:
    • infinite to make the animation loop forever.
    • A specific number (e.g., 3 for three repetitions).
  • direction: Specifies the direction of the animation. Possible values:
    • normal (default): The animation plays forward.
    • reverse: The animation plays backward.
    • alternate: The animation alternates between playing forward and backward.
    • alternate-reverse: Alternates, starting with playing backward.
  • fill-mode: Defines what styles are applied before and after the animation. Common values:
    • none (default): No styles are retained after the animation ends.
    • forwards: The element will keep the styles from the last keyframe (after the animation ends).
    • backwards: The element will apply styles from the first keyframe during the delay period.
    • both: Combines the effects of forwards and backwards.

Example of @keyframes and animation 


        
            @keyframes slide {
                0% {
                    transform: translateX(0);
                    background-color: red;
                }
                50% {
                    transform: translateX(100px);
                    background-color: yellow;
                }
                100% {
                    transform: translateX(200px);
                    background-color: green;
                }
            }
            
            div {
                width: 100px;
                height: 100px;
                background-color: red;
                animation: slide 4s ease-in-out infinite;
            }

In this example:

        The @keyframes rule defines how the element should move and change color from 0% (start) 
        to 100% (end).
        The div element will move from left to right over 4 seconds, 
        changing its background color along the way.
        The animation loops infinitely due to infinite.
